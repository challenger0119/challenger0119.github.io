Fibonacci

  • Fibonacci 数列的增长速度基本接近2的幂 一般来说是 Fn ~= 2^0.694n
  • 直接使用Fibonacci定义进行计算是效率很低的，因为 递归算法导致中间很多计算都是重复的 : 假设T(n) 是Fib的计算次数 那么根据fib的定义 —— T(n) = T(n-1) + T(n - 2) + 3 （3 为三次基本的计算）。这个公式本身又是一个Fib序列，所以其计算出的计算结果 也是指数的 （>= 2^0.694n）
  • 所以直接使用定义计算Fib几乎是无法计算的，考虑到中间的无用计算如果能存储起来直接使用，那么计算复杂度就是线性的，那么就有了 多项式计算方法 对比来看：
int fibonacci(int n){
    if(n == 0) return 0;
    if(n == 1) return 1;
    return fibonacci(n - 1) + fibonacci(n - 2);
}

int fibonacci_pro(int n){
    int *f = (int *)malloc(sizeof(int) * (n+1));
    for(int i = 0;i<=n;i++){
        if(i == 0){
            f[i] = 0;
        }else if(i == 1){
            f[i] = 1;
        }else{
            f[i] = f[i-1] + f[i-2];
        }
    }
    int result = f[n];
    free(f);
    return result;
}
  • 上面的算法复杂度忽略了一个维度：数字的位数。对于二进制的计算，每一次计算其实是一个位上的计算。所以实际的计算复杂度对于fib 就是 nFn 而 fib_pro 就是 n^2

大O符号

两个算法的运行时间分别为f(n) g(n)， f,g 都是从正整数到正实数的函数 如果存在一个常数c>0 是的 f(n) <= c · g(n) 那么 f = O(g) 即 f的增长速度慢于g 会被g超越

反过来说 g = Ω(f)

如果f = O(g) 的同时 g = O(f) 那么 f=θ(g)

支配原则（算法复杂度的大小比较） ：

  • 常系数可以忽略 14n^2 可变为 n^2
  • 当a > b 时候 n^a 支配 n^b 例如 n^2 支配 n
  • 指数项 支配 多项式 3^n 支配 n^5
  • 多项式支配 对数项 n 支配 log(n) ^ 3

logn复杂度的产生：log复杂度一般来说是应用了分治思想

以二分为例：如果一个数组长度是N，那么二分就是N/2 再二分就是 N/4最坏的情况分到 1 分的次数为x 则 N * (1/2)^x = 1 x = logN 底数2

因为通过矩阵计算Fib 时候 有 : FibAn = X^n FibA0 转为计算X^n 的问题

而计算X^n 时候可以使用二分 的思想 通过计算 (X^n/2) ^2，(X^n/4) ^4 ... 方法来进行矩阵乘法 所以进行矩阵乘法的次数是 O(logn)
