MHRSS 是一款运行于 鸿蒙（HarmonyOS） 系统的 RSS 阅读器客户端，通过 Google Reader API（GReader API） 协议与您自行选择的 RSS 服务提供方通信。

⚡ MHRSS 不运营任何 RSS 服务

我们仅作为客户端工具，为您连接兼容 GReader API 的第三方服务（如 The Old Reader、BazQux、Feedbin 等）。您的 RSS 数据完全存储在您所选的服务提供方，受其自身隐私政策约束。

我们收集的信息

  • 登录凭证： 您输入的服务器地址、用户名、密码或 API 令牌，仅存储在您的设备本地（使用鸿蒙系统安全存储能力），用于向您指定的 RSS 服务发起请求。MHRSS 没有服务器，不会上传或备份您的凭证。
  • 本地缓存： 订阅源列表、文章内容及阅读偏好，仅保存在您的设备上，用于离线阅读和提升体验。
  • 崩溃报告（可选）： 设备型号、鸿蒙系统版本、应用版本及错误堆栈，用于改进应用稳定性，不含任何个人身份信息或 RSS 内容。
✅ 登出即清除： 当您在应用内执行“登出”操作时，所有登录凭证及本地缓存数据将被立即清除。

我们如何保护您的数据

  • 本地优先： MHRSS 没有后端服务器，所有数据仅存于您的设备。
  • 加密传输： 所有 API 通信均使用 HTTPS/TLS 加密。
  • 安全存储： 凭证使用鸿蒙系统级安全存储能力（如 KeyStore）加密保存。
  • 权限最小化： 应用仅申请运行所必需的系统权限，不会获取与 RSS 阅读无关的权限。
  • 不与第三方共享： 我们不会出售、出租或向任何第三方透露您的个人信息。

您的权利

  • 您可随时在应用内“登出”，所有凭证和缓存数据将立即被清除。
  • 您也可直接卸载应用，删除所有本地数据。
  • 您可在所连接的 RSS 服务网站上管理或撤销 MHRSS 的访问权限。
  • 您可在鸿蒙系统设置中管理应用权限或关闭崩溃报告功能。

📌 您选择的第三方 RSS 服务（如 The Old Reader、BazQux、Feedbin 等）均有独立的隐私政策，请自行查阅以了解其数据处理方式。
